# phone_export — 功能规格说明

## 一、功能描述

从源 Excel 文件中读取指定的姓名列和手机号列，将数据转换后写入新的 Excel 文件。

转换规则：每个手机号独占一行；同一姓名对应多个手机号时，姓名单元格跨行合并，合并行数等于该姓名的手机号数量；手机号格式校验不通过的单元格在预览和输出文件中均标记红色背景；输出文件格式与源文件格式一致。

---

## 二、列映射配置（MVP 默认值，禁止在 UI 中修改）

| 字段名 | MVP 默认值 | 说明 |
|--------|-----------|------|
| `source_name_col` | `法定代表人` | 源表姓名列名 |
| `source_phone_col` | `有效手机号` | 源表手机号列名 |
| `target_name_col` | `姓` | 目标表姓名列名 |
| `target_phone_col` | `家庭手机` | 目标表手机号列名 |
| `target_name_index` | `0` | 姓名在目标模板中的列索引 |
| `target_phone_index` | `4` | 手机号在目标模板中的列索引 |

## 三、完整业务流程

### 步骤 1：用户选择源文件

- **执行模块**：`panel.py`（通过 `ui/components/file_picker.py`）
- **文件过滤**：仅展示 `.xls` 和 `.xlsx` 文件
- **成功输出**：源文件的绝对路径字符串
- **取消处理**：用户取消选择，不做任何操作，保持当前界面状态

### 步骤 2：检测格式，读取 Sheet 列表

- **执行模块**：`core/file_io/`（根据扩展名自动选择 Handler）
- **成功输出**：`List[str]`（文件中所有 Sheet 的名称列表，保持原始顺序）
- **失败处理**：抛出 `FileReadError` → `panel.py` 捕获 → 展示错误弹窗 + 记录 ERROR 日志 → **终止，等待用户重新选择文件**

### 步骤 3：用户选择 Sheet

- **执行模块**：`panel.py`（下拉框展示 Sheet 名称列表）
- **默认选中**：列表中的第一个 Sheet
- **成功输出**：用户选中的 Sheet 名称字符串

### 步骤 4：读取 Sheet 数据

- **执行模块**：`core/file_io/`
- **成功输出**：`SheetData` 对象（sheet_name + headers + rows）
- **失败处理**：抛出 `FileReadError` → `panel.py` 捕获 → 展示错误弹窗 + 记录 ERROR 日志 → **终止**

### 步骤 5：校验列名是否存在

- **执行模块**：`processor.py`
- **校验内容**：`ColumnMapping.source_name_col` 和 `ColumnMapping.source_phone_col` 必须同时存在于 `SheetData.headers`
- **成功输出**：无返回值（校验通过则继续）
- **失败处理**：抛出 `ColumnNotFoundError`（message 中包含缺失的列名和当前全部列名）→ `panel.py` 捕获 → 展示错误弹窗 + 记录 ERROR 日志 → **终止**

### 步骤 6：执行数据转换

- **执行模块**：`processor.py`
- **成功输出**：`PreviewData` 对象（含全量 `List[ExportRow]` + 校验摘要）
- **失败处理**：意外异常包装为 `DataProcessError` 抛出 → `panel.py` 捕获 → 展示错误弹窗 + 记录 ERROR 日志 → **终止**

### 步骤 7：展示预览面板

- **执行模块**：`panel.py`
- **行为规格**：见第六节「UI 行为规格 — 预览面板」
- **用户操作分支**：
  - 点击「确认导出」或「忽略并继续导出」→ 进入步骤 8
  - 点击「取消」→ 回到初始状态，不执行任何文件操作

### 步骤 8：用户选择输出路径

- **执行模块**：`panel.py`（调用系统文件保存对话框）
- **默认文件名**：`<源文件名（不含扩展名）>_<yyyyMMddHHmmss>.<原扩展名>`
- **默认保存路径**：源文件所在目录
- **成功输出**：用户确认的输出文件绝对路径字符串
- **取消处理**：用户取消，回到预览面板（不关闭预览，不清除数据）

### 步骤 9：写入新文件

- **执行模块**：`core/file_io/`（格式与源文件一致）
- **写入内容**：第 1 行为完整的 25 列 WPS 通讯录导入模板表头，第 2 行起为数据（姓名填入「姓」列，手机号填入「家庭手机」列，其余列留空）
- **合并规则**：`is_name_cell=True` 的行为合并起始行，向下合并 `merge_span` 行
- **样式规则**：`phone_valid=False` 的手机号单元格写入红色背景
- **失败处理**：抛出 `FileWriteError` → `panel.py` 捕获 → 展示错误弹窗 + 记录 ERROR 日志 → **终止**

### 步骤 10：展示成功信息

- **执行模块**：`panel.py`
- **展示内容**：`导出成功，共 {N} 行，文件已保存至：{输出路径}，日志见：{日志路径}`
- **记录日志**：INFO 级别，内容同上

## 四、数据转换规则

### 手机号分割流程

1. 取源行 `source_phone_col` 列的值（已转为字符串）
2. 按英文分号 `;` 分割，得到字符串列表
3. 对列表中每个元素执行 `strip()`（去除首尾空白字符）
4. 过滤掉 strip 后为空字符串的元素
5. 若过滤后列表为空，则视为该行手机号为空，生成一条 `phone=""` 的 ExportRow

### merge_span 和 is_name_cell 计算规则

| 场景 | 生成 ExportRow 数量 | merge_span | is_name_cell |
|------|---------------------|------------|--------------|
| 手机号分割后有 N 个有效段（N≥2） | N 条 | 所有条均为 N | 第 1 条为 True，其余为 False |
| 手机号分割后有 1 个有效段 | 1 条 | 1 | True |
| 手机号为空（无有效段） | 1 条 | 1 | True |

### source_row_index 计算规则

- 从 1 开始计数，对应源表数据行的序号（表头行不计，第一条数据行为 1）
- 同一源行拆分出的多条 ExportRow 共享**相同**的 `source_row_index`

## 五、手机号校验规则

校验在 `processor.py` 中执行，**校验前先对手机号执行 `strip()`**。

### 规则一：跳过校验（优先判断）

以 `+` 开头的手机号 → `phone_valid = True`，跳过后续规则，原样写入。

### 规则二：大陆手机号合法性校验

必须**同时满足**以下全部条件才判定为合法（`phone_valid = True`）：

| 序号 | 条件 |
|------|------|
| 1 | 长度为 11 位 |
| 2 | 全部为数字字符（`str.isdigit()` 返回 True） |
| 3 | 第 1 位（index 0）为 `1` |
| 4 | 第 2 位（index 1）在 `['3','4','5','6','7','8','9']` 中 |

不满足任意一条 → `phone_valid = False`。

### 空手机号（`phone=""`）

固定 `phone_valid = True`，不参与校验，不计入 `invalid_count`。

### invalid_summary 条目格式

```
第 {source_row_index} 行：{phone} 不是合法手机号
```

## 六、UI 行为规格

### 预览面板

| 项目 | 规格 |
|------|------|
| 默认展示行数 | 30 行 |
| 展开按钮出现条件 | 转换结果总行数 > 30 |
| 展开按钮文案 | `展开查看全部 {N} 行` |
| 折叠按钮文案 | `收起` |
| 校验不通过单元格样式 | 红色背景（`#FF0000`） |
| 顶部汇总文案（无异常） | `共 {N} 行，手机号格式均合法` |
| 顶部汇总文案（有异常） | `共 {N} 行，其中 {M} 个手机号格式异常` |
| 警告横幅出现条件 | `invalid_count > 0` |
| 警告横幅内容 | 逐条展示 `invalid_summary` 中的每一条错误描述 |
| 按钮组（无异常时） | 「确认导出」「取消」 |
| 按钮组（有异常时） | 「忽略并继续导出」「取消」 |

### 输出文件

| 项目 | 规格 |
|------|------|
| 默认文件名格式 | `<源文件名（不含扩展名）>_<yyyyMMddHHmmss>.<原扩展名>` |
| 默认保存路径 | 源文件所在目录 |
| 输出格式 | 与源文件完全一致（xls→xls，xlsx→xlsx） |
| 首行内容 | 完整的 25 列表头（WPS 通讯录导入模板），姓名→「姓」列（col 0），手机号→「家庭手机」列（col 4） |
| 姓名单元格 | `is_name_cell=True` 的行在「姓」列写入姓名并向下合并 `merge_span` 行 |
| `is_name_cell=False` 的姓名单元格 | 属于合并区域，不单独写入任何值（该位置在合并后可留空或填入原始值，但合并区域只保留首行值） |
| 异常手机号单元格样式 | 红色背景（`#FF0000`），与预览面板一致 |
| 合法手机号和空手机号单元格 | 无特殊样式，保持默认 |

## 七、边界情况处理表

| 场景 | 处理方式 |
|------|---------|
| 某行姓名（`source_name_col`）为空 | 正常处理，名字单元格写入空字符串，不报错 |
| 某行手机号（`source_phone_col`）为空 | 生成 1 条 `phone=""` 的 ExportRow，`phone_valid=True`，不报错 |
| 某行手机号全部格式不合法 | 全部标红，`merge_span` 照常计算，允许用户选择「忽略并继续导出」 |
| 源 Sheet 只有表头无数据行 | `processor.py` 返回 `rows=[]` 的 `PreviewData`，`panel.py` 展示提示「未找到可导出的数据」，不进入预览步骤 |
| 手机号分号前后有空格（如 `138xxxx ; 139xxxx`） | 每段 `strip()` 后再校验，不视为非法字符 |
| 手机号列有连续分号（如 `138xxxx;;139xxxx`） | 中间空段 `strip()` 后为空字符串，过滤掉，不生成对应 ExportRow |
| 手机号以 `+` 开头 | 不校验，`phone_valid=True`，原样写入 |
| 手机号带前缀 `+86`（如 `+8613800000000`） | 以 `+` 开头，不校验，原样写入 |

## 八、不在本功能范围内的事项

以下事项明确不做，禁止 Agent 自行实现：

- 不校验姓名格式
- 不对数据去重（同一姓名多次出现，按多条独立数据处理）
- 不对数据排序
- 不修改源文件
- 不支持同时处理多个 Sheet（只处理用户在 UI 中选定的单个 Sheet）
- 不支持向已存在的目标文件追加写入（每次生成全新文件）
- 不支持在 UI 中修改列映射配置（MVP 阶段）
